The pipeline

8 stages from
question to answer

Every query travels through a multi-stage retrieval and synthesis pipeline — not just a single LLM call.

1

Natural language question

You ask anything, in plain English, typos and all. The system handles ambiguity in the next step.

2

Query expansion

A lightweight AI rewrites and expands your question — fixing typos, adding synonyms, making implicit context explicit — to maximise how much relevant content gets retrieved.

Llama 3.1 8B · Groq
3

Semantic embedding

Your expanded question is converted into a 1024-dimensional vector representing its meaning — not just its keywords. This enables matching even when the wording differs from the documentation.

Jina Embeddings v3 · 1024-dim cosine
4

Intent routing

The system classifies your original query to decide the best search strategy. Named components → filtered dual-path search. Conceptual questions → unified search across all 751 vectors.

Regex classifier · 2 routes
5

Vector search

Qdrant Cloud retrieves the passages whose meaning is most similar to your question, using cosine similarity. Up to 10 candidates are returned.

Qdrant Cloud · 751 vectors
6

Neural reranking

A cross-encoder model reads your question and each candidate passage together — far more accurate than initial similarity matching — and selects the top 5 most relevant passages.

Jina Reranker v2 · top_n=5
7

Grounded synthesis

A large language model generates an answer using only the 5 retrieved passages as source material. It is explicitly instructed to cite passage numbers inline and omit anything not traceable word-for-word.

Qwen3-32B · Groq · /no_think
8

Cited answer

You receive a response with inline citations [1][2] and a Sources section listing originating filenames. Every claim is verifiable.

Run it yourself

Import this pipeline
over your own docs

The Dify DSL export is included in this repo with API keys removed. You can have your own version running in under an hour — all on free tiers.

1
Sign up for Dify Cloud, Qdrant Cloud, Groq, and Jina AI — all free
2
Create a Qdrant collection with named dense vectors (1024-dim, cosine)
3
Ingest your documents using the payload schema in the import guide
4
Import the DSL from dify-workflow/talk-to-db-pro.yml into a new Dify Chatflow
5
Set 3 environment variables in Dify: JINA_API_KEY, QDRANT_API_KEY, QDRANT_POINTS_URL
6
Publish and chat — your docs are now conversational
